## УИ‑ТЭЦ портал

Внутренний web-портал для смены и подразделений ТЭЦ. Приложение построено на Vite + React 19 и Supabase и даёт сотрудникам единое окно для оперативной информации, журналов и календаря смен.

### Основные возможности
- Лэндинг смены с быстрыми метриками, идеями и блоком связи профиля с кадровой записью.
- Аутентификация через Supabase Auth, защищённые разделы и редирект на страницу входа.
- Навигация по подразделениям (КТЦ, химцех, электроцех, ЦТАиИ, топливоподача) и секциям «Персонал / Оборудование / Документация».
- Раздел КТЦ → Документация: журнал распоряжений/ведомостей с созданием записей, отметкой ознакомления, счётчиком новых и фильтрами по типам.
- Раздел КТЦ → Персонал: интерактивный календарь смен (по сотрудникам и датам), пресеты смен/статусов, массовое проставление по диапазону, фильтры по категории/отделению/должности, закрепление и скрытие сотрудников.
- Профиль пользователя: привязка профиля к записи сотрудника, редактирование ФИО/контактов/детей, отображение подразделения/должности, проверка членства в профсоюзе.
- Вспомогательные страницы: реестр оборудования, состав смены, новости/услуги профсоюза, виджет погоды по Усть-Илимску.

### Технологический стек
- React 19, React Router 7, @tanstack/react-query
- Vite 7, TailwindCSS
- Supabase (Auth + Postgres + RLS), @supabase/supabase-js

### Быстрый старт
1) Установите зависимости: `npm install`
2) Скопируйте `.env.example` (создайте, если нужно) в `.env.local` и задайте ключи:
   - `VITE_SUPABASE_URL=<ваш Supabase URL>`
   - `VITE_SUPABASE_ANON_KEY=<anon/public ключ>`
3) Запустите dev-сервер: `npm run dev`
4) Сборка: `npm run build`, предпросмотр: `npm run preview`, линт: `npm run lint`

### Ожидаемая схема данных (Supabase)
- `profiles` (id, employee_id, updated_at)
- `employees` (id, first_name, last_name, middle_name, position_id, phone, birth_date, union_member, auth_user_id)
- `positions` (id, name, departament_name, devision_name, type, sort_weight)
- `children` (id, id_employees, first_name, last_name, middle_name, birth_date)
- `equipment`, 
- `journals`, `entries`, `entry_receipts`, `entry_reads`
- `schedule`, `schedule_overrides`, `shift_templates`

RLS/политики должны разрешать чтение/запись под нужные роли. Без валидных переменных окружения провайдер выводит подсказку о настройке Supabase.
